# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## данные

в приложении используются данные о товарах, данные покупателя, данные для отправки заказа на сервер и данные, которые приходят от сервера в ответ на запросы.

### тип `TPayment`

тип описывает доступные способы оплаты заказа. значение может быть только одним из двух вариантов: `card` или `cash`.

```ts
export type TPayment = 'card' | 'cash';
```

### интерфейс `IProduct`

интерфейс описывает товар, который отображается в каталоге, может быть открыт для подробного просмотра и добавлен в корзину.

```ts
export interface IProduct {
    id: string;
    description: string;
    image: string;
    title: string;
    category: string;
    price: number | null;
}
```

поля интерфейса:

- `id: string` — уникальный идентификатор товара;
- `description: string` — описание товара;
- `image: string` — путь к изображению товара;
- `title: string` — название товара;
- `category: string` — категория товара;
- `price: number | null` — цена товара. если значение равно `null`, товар недоступен для покупки.

### интерфейс `IBuyer`

интерфейс описывает данные покупателя, необходимые для оформления заказа.

```ts
export interface IBuyer {
    payment: TPayment;
    email: string;
    phone: string;
    address: string;
}
```

поля интерфейса:

- `payment: TPayment` — выбранный способ оплаты;
- `email: string` — электронная почта покупателя;
- `phone: string` — телефон покупателя;
- `address: string` — адрес доставки.

### интерфейс `IProductsResponse`

интерфейс описывает ответ сервера при получении каталога товаров.

```ts
export interface IProductsResponse {
    total: number;
    items: IProduct[];
}
```

поля интерфейса:

- `total: number` — общее количество товаров;
- `items: IProduct[]` — массив товаров.

### интерфейс `IOrderRequest`

интерфейс описывает объект, который отправляется на сервер при оформлении заказа.

```ts
export interface IOrderRequest extends IBuyer {
    total: number;
    items: string[];
}
```

поля интерфейса:

- `payment: TPayment` — выбранный способ оплаты;
- `email: string` — электронная почта покупателя;
- `phone: string` — телефон покупателя;
- `address: string` — адрес доставки;
- `total: number` — итоговая стоимость заказа;
- `items: string[]` — массив идентификаторов товаров в заказе.

### интерфейс `IOrderResponse`

интерфейс описывает ответ сервера после успешного оформления заказа.

```ts
export interface IOrderResponse {
    id: string;
    total: number;
}
```

поля интерфейса:

- `id: string` — идентификатор созданного заказа;
- `total: number` — итоговая стоимость заказа.

## модели данных

### класс `Products`

класс отвечает за хранение каталога товаров и товара, выбранного для подробного просмотра. модель не работает с DOM-элементами и не отвечает за отображение данных на странице.

конструктор класса не принимает параметров.

поля класса:

- `items: IProduct[]` — массив всех товаров каталога;
- `selectedProduct: IProduct | null` — товар, выбранный для подробного просмотра. если товар не выбран, хранится значение `null`.

методы класса:

#### `setItems(items: IProduct[]): void`

метод сохраняет массив товаров в модель.

параметры:

- `items: IProduct[]` — массив товаров, полученный из тестовых данных или с сервера.

возвращаемое значение: `void`.

#### `getItems(): IProduct[]`

метод возвращает массив всех товаров, сохранённых в модели.

возвращаемое значение:

- `IProduct[]` — массив товаров.

#### `getProduct(id: string): IProduct | undefined`

метод ищет товар в каталоге по его идентификатору.

параметры:

- `id: string` — идентификатор товара.

возвращаемое значение:

- `IProduct | undefined` — найденный товар или `undefined`, если товара с таким идентификатором нет.

#### `setSelectedProduct(product: IProduct): void`

метод сохраняет товар, выбранный для подробного просмотра.

параметры:

- `product: IProduct` — выбранный товар.

возвращаемое значение: `void`.

#### `getSelectedProduct(): IProduct | null`

метод возвращает товар, выбранный для подробного просмотра.

возвращаемое значение:

- `IProduct | null` — выбранный товар или `null`, если товар не выбран.

### класс `Basket`

класс отвечает за хранение товаров, которые покупатель добавил в корзину. модель не работает с DOM-элементами и не отвечает за отображение корзины на странице.

конструктор класса не принимает параметров.

поля класса:

- `items: IProduct[]` — массив товаров, добавленных в корзину.

методы класса:

#### `getItems(): IProduct[]`

метод возвращает массив товаров, которые находятся в корзине.

возвращаемое значение:

- `IProduct[]` — массив товаров в корзине.

#### `addItem(product: IProduct): void`

метод добавляет товар в корзину.

параметры:

- `product: IProduct` — товар, который нужно добавить в корзину.

возвращаемое значение: `void`.

#### `removeItem(product: IProduct): void`

метод удаляет товар из корзины по его идентификатору.

параметры:

- `product: IProduct` — товар, который нужно удалить из корзины.

возвращаемое значение: `void`.

#### `clear(): void`

метод полностью очищает корзину.

возвращаемое значение: `void`.

#### `getTotalPrice(): number`

метод возвращает общую стоимость товаров в корзине. если у товара цена равна `null`, при подсчёте она считается как `0`.

возвращаемое значение:

- `number` — итоговая стоимость товаров в корзине.

#### `getCount(): number`

метод возвращает количество товаров в корзине.

возвращаемое значение:

- `number` — количество товаров в корзине.

#### `hasItem(id: string): boolean`

метод проверяет, находится ли товар в корзине.

параметры:

- `id: string` — идентификатор товара.

возвращаемое значение:

- `boolean` — `true`, если товар есть в корзине, и `false`, если товара в корзине нет.

### класс `Buyer`

класс отвечает за хранение данных покупателя, необходимых для оформления заказа. модель позволяет сохранять данные частями, получать все сохранённые данные, очищать их и проверять заполненность полей.

конструктор класса не принимает параметров.

поля класса:

- `payment: IBuyer['payment'] | null` — выбранный способ оплаты. если способ оплаты не выбран, хранится значение `null`;
- `email: string` — электронная почта покупателя;
- `phone: string` — телефон покупателя;
- `address: string` — адрес доставки.

методы класса:

#### `setData(data: Partial<IBuyer>): void`

метод сохраняет данные покупателя в модель. можно передать только часть данных, например только адрес или только телефон. уже сохранённые значения при этом не удаляются.

параметры:

- `data: Partial<IBuyer>` — объект с одним или несколькими полями покупателя.

возвращаемое значение: `void`.

#### `getData(): IBuyer`

метод возвращает все данные покупателя.

возвращаемое значение:

- `IBuyer` — объект с данными покупателя.

#### `clear(): void`

метод очищает данные покупателя.

возвращаемое значение: `void`.

#### `validate(): TBuyerErrors`

метод проверяет заполненность полей покупателя и возвращает объект с ошибками. если поле заполнено корректно, ошибка для него не добавляется.

возвращаемое значение:

- `TBuyerErrors` — объект, в котором ключами являются поля покупателя, а значениями — тексты ошибок.

пример объекта ошибок:

```ts
{
    payment: 'не выбран способ оплаты',
    email: 'укажите email',
}
```

## слой коммуникации

### класс `WebLarekApi`

класс отвечает за взаимодействие приложения с сервером. он использует объект, соответствующий интерфейсу `IApi`, и через него выполняет запросы к api. класс не хранит данные приложения и не работает с DOM-элементами.

конструктор класса:

```ts
constructor(api: IApi)
```

параметры конструктора:

- `api: IApi` — объект api-клиента, который содержит методы `get` и `post`.

поля класса:

- `api: IApi` — экземпляр api-клиента, используемый для отправки запросов на сервер.

методы класса:

#### `getProducts(): Promise<IProductsResponse>`

метод выполняет `GET`-запрос на эндпоинт `/product/` и получает каталог товаров с сервера.

возвращаемое значение:

- `Promise<IProductsResponse>` — промис с объектом, в котором находится общее количество товаров и массив товаров.

#### `orderProducts(order: IOrderRequest): Promise<IOrderResponse>`

метод выполняет `POST`-запрос на эндпоинт `/order/` и отправляет данные заказа на сервер.

параметры:

- `order: IOrderRequest` — объект с данными покупателя, итоговой стоимостью заказа и массивом идентификаторов товаров.

возвращаемое значение:

- `Promise<IOrderResponse>` — промис с объектом, подтверждающим успешное оформление заказа.
